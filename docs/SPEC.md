# 💾 projnew: プロジェクトテンプレートクローンツール 仕様書 (SPEC.md)

## 1. 概要

`projnew` は、Gitリポジトリをテンプレートとして使用し、新しいプロジェクトを素早く作成するためのコマンドラインツールです。設定ファイルに基づく柔軟なファイル内容置換と、クローン後の自動セットアップ（`postCloneActions`）をサポートします。

## 2. コマンドと機能

### 2.1. プロジェクト作成 (メイン機能)
```bash
projnew <テンプレートID> <プロジェクト名>
```

  * **機能:** 指定されたIDのテンプレートをクローンし、プロジェクト名でディレクトリを作成、定義された処理を実行します。
  * **例:** `projnew electron my-new-app`

### 2.2. テンプレート一覧表示

```bash
projnew list
```

  * **機能:** 定義ファイルに登録されている全てのテンプレートIDとDescriptionを一覧表示します。

### 2.3. 設定ファイルの強制生成

```bash
projnew --generate-config / -g
```

  * **機能:** テンプレート定義ファイル (`projnew.templates.json`) を、グローバル設定パス (`$HOME/.projnew/`) に初期サンプルファイルとして強制的に生成します。
  * **衝突解決:** 既にグローバル設定パスに定義ファイルが存在する場合、既存ファイルを上書きせず、連番サフィックス（例: `projnew.templates.json.1`）を付けて新規ファイルとして生成します。ツールは生成後、直ちに終了します。

## 3\. テンプレート定義ファイル (`projnew.templates.json`)

### 3.1. 配置場所 (ポータビリティ優先の探索順序)

設定ファイルは、以下の優先順位で探索されます。

1.  **ポータブルパス (最優先):** `projnew` 実行ファイルと同じディレクトリ直下。
2.  **グローバルパス (フォールバック):** ユーザーのホームディレクトリ内の設定用隠しディレクトリ (`$HOME/.projnew/projnew.templates.json`)。

### 3.2. ファイル生成と初回起動処理

  * **自動生成:** 上記1, 2のいずれのパスにもファイルが存在しない場合、グローバルパス (`$HOME/.projnew/`) に初期サンプルファイルが**自動的に生成**されます。この自動生成が行われた場合、該当の起動セッションは「初回起動」と見なされます。
  * **初期ファイルの内容:** 必須要素を全て含み、`sourceUrl`にはクローンできない安全なダミーURL (`https://example.com/...`) を設定します。
  * **JSONコメント対応:** デフォルトでコメントがあるとデシリアライズに失敗するため、デシリアライズ処理には `JsonCommentHandling.Skip` を設定し、コメントを許可します。

### 3.3. 定義スキーマ

| フィールド | 型 | 必須 | 説明 |
| :--- | :--- | :--- | :--- |
| `id` | String | 必須 | テンプレートを一意に識別するID。コマンドライン引数で使用。 |
| `description` | String | 必須 | テンプレートの簡単な説明。`projnew list`で使用。 |
| `sourceType` | String | 必須 | ソースの種類。現在は `"git"` のみサポート。 |
| `sourceUrl` | String | 必須 | クローン対象のGitリポジトリURL。 |
| `defaultBranch` | String | 任意 | クローンするブランチ名。未指定の場合はリポジトリのデフォルトブランチを使用。 |
| `fileReplacements` | Array | 任意 | クローン後にファイル内のプレースホルダを置換する処理のリスト。 |
| `postCloneActions` | Array | 任意 | クローン後に実行するシェルコマンドのリスト。 |

#### 3.3.1. `fileReplacements` スキーマ

| フィールド | 型 | 必須 | 説明 |
| :--- | :--- | :--- | :--- |
| `fileName` | String | 必須 | 置換対象のファイル名（クローンルートからの相対パス）。 |
| `placeholder` | String | 必須 | ファイル内で置換される文字列（プレースホルダ）。 |
| `replacementValue` | String | 必須 | プレースホルダと置き換える新しい値。特殊な変数 `{PROJECT_NAME}` や `{CURRENT_YEAR}` をサポート。 |

## 4\. クローンと後処理の実装詳細

### 4.1. Git履歴の処理

  * **動作:** リポジトリは全履歴をクローンせず、指定されたブランチの**最新コミットのみを浅くクローン**（Shallow Clone）します。

### 4.2. 後処理コマンド (`postCloneActions`)

  * **実行環境の確定:** C\#の `System.Runtime.InteropServices.RuntimeInformation` を使用し、実行環境を判別します。

      * **Windows:** `cmd.exe /c "{コマンド}"` を使用してコマンドを実行。
      * **macOS/Linux:** `/bin/bash -c "{コマンド}"` を使用してコマンドを実行。
      * 外部プロセス起動には `System.Diagnostics.Process` クラスを使用し、標準入出力は非同期でホストコンソールにリダイレクトします。

  * **セキュリティ警告 (初回起動時):**

      * 定義ファイルがツールによって**自動生成**された場合、`postCloneActions` の実行前に、ユーザーに対して「**このコマンドは自己責任で実行してください。続行しますか？ (Y/N)**」という形式のセキュリティ警告と確認を行います。

## 5\. 使用技術（C\# .NET 8）

  * **言語:** C\# (.NET 8)
  * **コマンドライン解析:** `System.CommandLine` ライブラリ
  * **JSON処理:** `System.Text.Json`
  * **外部プロセス:** `System.Diagnostics.Process`
