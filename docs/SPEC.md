# 📄 CLIツール『projnew』設計ドキュメント骨子

## 1\. はじめに (Introduction)

### 1.1 目的

`projnew` は、ユーザー定義のGitテンプレートリポジトリを簡単にクローンし、新しいプロジェクト名への置換、および依存関係のインストール（`npm install`など）を自動で行う、クロスプラットフォームな CLI ツールである。目指すは、C\#における `dotnet new` のようなシンプルで拡張性の高いプロジェクト初期化体験の提供である。

### 1.2 スコープ

本ドキュメントは、ツールのコマンド仕様、テンプレート定義形式、および実装に必要な技術的・非機能的要件を定義する。

## 2\. 機能要件 (Functional Requirements)

### 2.1 コマンドラインインターフェース (CLI)

CLI名は **`projnew`** とする。

| 構文 | 機能 | 備考 |
| :--- | :--- | :--- |
| `projnew [ID] [NAME]` | テンプレートのクローンとセットアップ（メイン機能） | `ID` は `projnew.templates.json` に定義される。`NAME` は作成されるディレクトリ名。 |
| `projnew list` | 定義済みテンプレートの一覧表示 | エイリアスとして `projnew ls` を許容する。 |

### 2.2 オプション

| オプション | エイリアス | 機能 | 動作 |
| :--- | :--- | :--- | :--- |
| `--keep-history` | `-k`, `--keep-git` | クローン元のGit履歴を保持する | デフォルト（オプションなし）では `.git` ディレクトリを削除し、新規 `git init` を実行する。 |

## 3\. テンプレート定義仕様 (Template Definition Specification)

### 3.1 定義ファイル

  * **ファイル名:** `projnew.templates.json`
  * **配置場所:** `projnew` 実行ファイル（またはグローバルインストールされた場所）と同一のディレクトリ。

### 3.2 JSONスキーマ

ルートは `{ "templates": [ ... ] }` の形式を持つ。

```json
{
  "templates": [
    {
      "id": "string",             // 必須: コマンドで指定する一意のID
      "description": "string",    // 必須: 'projnew list' で表示される説明
      "sourceType": "string",     // 必須: 現在は "git" のみサポート
      "sourceUrl": "string",      // 必須: クローン元GitリポジトリのURL
      "defaultBranch": "string",  // 任意: クローンするブランチ名 (デフォルトは "main")
      
      "fileReplacements": [ ... ],// 任意: ファイル内容の置換ルール
      "postCloneActions": [ ... ] // 任意: クローン後に実行するシェルコマンド
    }
  ]
}
```

### 3.3 ファイル置換ルール (`fileReplacements`)

3つの要素を持つオブジェクトの配列。

| フィールド | 型 | 説明 |
| :--- | :--- | :--- |
| `fileName` | string | 置換対象のファイル名（プロジェクトルートからの相対パス） |
| `placeholder` | string | ファイル内で検索し、置き換える対象の文字列 |
| `replacementValue`| string | 置き換え後の値。内部プレースホルダを使用可能。 |

#### 内部プレースホルダ

  * `{PROJECT_NAME}`: コマンドで指定された新しいプロジェクト名 (`NAME`)
  * `{CURRENT_YEAR}`: ツール実行時の西暦年号

## 4\. 処理フロー詳細 (Execution Flow)

`projnew [ID] [NAME]` 実行時の主要フェーズは以下の通り。

### 4.1 フェーズ 1: 初期化とクローン

1.  **JSON読み込み:** `projnew.templates.json` を読み込み、`ID` に一致するテンプレート定義を検索する。
2.  **Git Clone:** `sourceUrl` を使用して `git clone` を実行し、`NAME` のディレクトリを作成する。
      * **フィードバック:** Gitの出力を非同期でリアルタイムにコンソールにストリームする。
3.  **Git Cleanup:** `--keep-history` オプションがない場合、`.git` ディレクトリを削除し、新規に `git init` を実行する。

### 4.2 フェーズ 2: ファイル内容の置換

1.  `fileReplacements` リストをループする。
2.  各置換ルールに基づき、`placeholder` を決定された `replacementValue`（内部プレースホルダを解決したもの）に置き換える。
      * **技術的詳細:** 置換処理は、ファイルをメモリに一括読み込みし、`string.Replace` 等で効率的に行う。

### 4.3 フェーズ 3: 後処理コマンドの実行

1.  `postCloneActions` リストの実行前に、**セキュリティ警告メッセージ**を表示し、ユーザーの同意を得る。
2.  同意が得られた場合、リスト内の各コマンドを順に、新しいプロジェクトディレクトリをカレントディレクトリとして実行する。
      * **外部プロセス:** C\#の `Process` クラスを使用し、OSに応じたシェルを介して実行する。

## 5\. 非機能要件 (Non-Functional Requirements)

### 5.1 互換性と依存関係

  * **フレームワーク:** **.NET 8 以降**を使用する。
  * **ターゲットOS:** **Windows 11** を必須サポートとし、macOS/Linux は動作可能とする（動作保証外）。
  * **必須依存:** ユーザー環境に **Git コマンドラインツール**がインストールされ、`PATH` に登録されていることを前提とする。

### 5.2 外部プロセス実行戦略

  * **OS判定:** `System.Runtime.InteropServices.RuntimeInformation` を使用して OS を判定する。
  * **シェルラッパー:**
      * **Windows:** `cmd.exe /c "[COMMAND]"`
      * **Unix系 (macOS/Linux):** `/bin/bash -c "[COMMAND]"`
  * **入出力:** 標準出力/エラー出力は `RedirectStandardOutput`/`RedirectStandardError` を設定し、**非同期イベントハンドラ**を介してリアルタイムにコンソールへストリームする。

### 5.3 エラーハンドリング

| エラーの種類 | 処理 | メッセージ |
| :--- | :--- | :--- |
| **Git/外部コマンド失敗** | 即座に処理を中断。 | 外部コマンドの終了コードと、STDOUT/STDERR の出力をそのまま表示する。 |
| **JSON構造不正** | 処理を中断。 | 必要な構造、必須フィールド、使用例を含む**英語のガイダンス**メッセージを表示する。 |

### 5.4 セキュリティ

  * **責任:** `postCloneActions` の内容はツールの保証外とし、**ユーザーの自己責任**とする。
  * **警告:** `postCloneActions` 実行直前に、**外部コマンド実行のリスクを英語で警告**し、ユーザーに続行の同意を求める。対話型認証のハンドリングは実装しない。
